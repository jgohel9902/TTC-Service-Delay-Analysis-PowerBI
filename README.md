# TTC Service Delay Analysis | Power BI Geospatial Dashboard

## The Problem
Toronto's TTC generates thousands of service incidents monthly.
Raw delay data exists — but without structure, it can't answer
the questions operations teams actually need:
Which routes are highest risk? Where are the congestion corridors?
Is severity growing or shrinking over time?

## What This Project Does
Transforms raw TTC delay data into a 3-layer executive intelligence
dashboard built in Power BI — covering KPI monitoring, route risk
classification, and 3D geospatial hotspot mapping.

## Dashboard Structure

### Page 1 — Executive Overview
- Total Delay Minutes, Incidents, Average Delay, Severity %
- Top 10 Routes by Total Delay Impact
- Monthly Delay Trend

### Page 2 — Route Performance Intelligence
- Risk Quadrant: Incidents vs Average Delay Severity
- Top Routes by Incident Count and Delay Severity
- Route Rank Movement Over Time (Ribbon Chart)

### Page 3 — Geospatial Hotspot Analysis
- 3D Azure Maps Delay Intensity Visualization
- Directional Delay Distribution by Day
- Operational Congestion Corridor Identification

## Key Insight
High-incident routes are not always the highest-severity routes.
Frequency and severity tell different stories —
and operational decisions require understanding both.
